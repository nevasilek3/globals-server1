# 🚀 Развертывание на Render.com с SQLite

## ✅ Преимущества SQLite на Render

- ✅ Бесплатно (не нужна отдельная БД)
- ✅ Все данные сохраняются в файле `bot_data.db`
- ✅ Данные не теряются при перезагрузке
- ✅ Простая настройка
- ✅ Подходит для небольших проектов

## 📋 Шаги развертывания

### 1. Подготовка репозитория

Убедитесь, что у вас есть:
- `Server/` папка с ботом
- `requirements.txt` с зависимостями
- `.env.example` с переменными окружения

### 2. Создание Web Service на Render

1. Перейдите на [render.com](https://render.com)
2. Нажмите **"New +"** → **"Web Service"**
3. Подключите GitHub репозиторий

### 3. Настройка Web Service

**Основные параметры:**
- **Name**: `telegram-bot` (или ваше имя)
- **Environment**: Python 3
- **Region**: Выберите ближайший регион
- **Branch**: main (или ваша ветка)

**Build Command:**
```bash
pip install -r Server/requirements.txt
```

**Start Command:**
```bash
cd Server && python main.py
```

### 4. Переменные окружения

Добавьте в разделе "Environment":

| Переменная | Значение |
|-----------|---------|
| `BOT_TOKEN` | Ваш Telegram токен |
| `RENDER_EXTERNAL_URL` | `https://your-app-name.onrender.com` |
| `DATABASE_URL` | `sqlite:///./bot_data.db` |
| `SERVER_PORT` | `8000` |

### 5. Развертывание

1. Нажмите **"Create Web Service"**
2. Ждите завершения сборки (2-3 минуты)
3. Проверьте логи на наличие ошибок

## 🔍 Проверка статуса

### Проверить здоровье сервера:
```bash

```

## 📊 Где хранятся данные

Все данные хранятся в файле `bot_data.db` в корне приложения на Render.

**Важно:** Файл сохраняется между перезагрузками, но если вы удалите сервис, данные будут потеряны.

## 💾 Резервная копия данных

### Скачать базу данных:

1. Подключитесь к Render Shell
2. Выполните:
```bash
cat bot_data.db | base64
```
3. Скопируйте вывод и сохраните в файл

### Восстановить базу данных:

1. Загрузите файл на Render
2. Выполните:
```bash
echo "base64_content" | base64 -d > bot_data.db
```

## 🐛 Решение проблем

### Ошибка: "ModuleNotFoundError: No module named 'psycopg2'"

**Решение:** Убедитесь, что `requirements.txt` не содержит `psycopg2-binary`. Используйте SQLite.

### Ошибка: "Database is locked"

**Решение:** SQLite имеет ограничения при одновременном доступе. Если это происходит часто, рассмотрите использование PostgreSQL на другом хостинге.

### Бот не отвечает

1. Проверьте логи на Render
2. Убедитесь, что `BOT_TOKEN` правильный
3. Проверьте, что сервер запущен

### Данные не сохраняются

1. Проверьте права доступа к файлу `bot_data.db`
2. Убедитесь, что БД инициализирована
3. Проверьте логи на ошибки

## 📈 Масштабирование

Если проект растет и SQLite становится узким местом:

1. Создайте PostgreSQL на [Neon.tech](https://neon.tech) (бесплатно)
2. Обновите `DATABASE_URL` в переменных окружения
3. Перезагрузите сервис

## 🔐 Безопасность

- ✅ Пароли хешируются (bcrypt)
- ✅ Используйте HTTPS (Render предоставляет автоматически)
- ✅ Никогда не коммитьте `.env` файл
- ✅ Используйте `.gitignore` для `bot_data.db`
